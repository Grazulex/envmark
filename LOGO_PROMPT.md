# EnvMark Logo - Prompt pour Grok/Aurora

## Prompt Principal

```
Design a modern, minimalist logo for "EnvMark" - a developer tool for managing environment variables (.env files) using Git.

Style requirements:
- Clean, flat design with no gradients
- Primary color: Emerald green (#10B981)
- Secondary accent: Indigo (#6366F1)
- Suitable for dark and light backgrounds
- Must work as a small favicon (32x32) and large banner

Concept elements to incorporate:
- Environment/nature element (leaf, tree, or eco symbol) representing ".env"
- Git/version control element (branch, merge, or commit node)
- Security/lock element (subtle, representing secure secret management)
- Bookmark or checkmark representing the "Mark" suffix

The logo should feel:
- Professional and trustworthy
- Modern and developer-focused
- Part of a suite (similar style to Backmark, Stackmark, Shipmark)

Avoid:
- Overly complex designs
- Realistic illustrations
- Too many colors
- Text in the icon version
```

## Variations demandées

### 1. Icon seul (carré)
```
Create a square icon version of the EnvMark logo:
- Simple geometric shapes
- Works at 32x32, 64x64, 128x128, 512x512
- Emerald green (#10B981) as primary
- Transparent background
- Single recognizable symbol combining: environment + git + security
```

### 2. Logo horizontal (avec texte)
```
Create a horizontal logo with icon + "EnvMark" text:
- Icon on the left, text on the right
- Font: Modern sans-serif (similar to Inter, Geist, or SF Pro)
- "Env" in emerald green, "Mark" in dark gray (#1F2937)
- Suitable for website header and documentation
```

### 3. Badge/Shield version
```
Create a badge-style logo for GitHub README:
- Shield or badge shape
- "EnvMark" text integrated
- Version number placeholder area
- Emerald green (#10B981) background
- White text
```

## Concepts suggérés

### Concept A: Leaf + Branch
- Une feuille stylisée (env = environment)
- Avec une branche Git intégrée dans la tige
- Forme simple et reconnaissable

### Concept B: Lock + Variable
- Un cadenas ouvert/fermé
- Avec des symboles de variables `{ }` ou `$`
- Représente la sécurité des secrets

### Concept C: Bookmark + Dot
- Un signet (bookmark pour "Mark")
- Avec un point vert (comme le point dans .env)
- Clean et minimaliste

### Concept D: Git Node + Shield
- Un nœud de commit Git
- Entouré d'un bouclier de protection
- Moderne et tech-focused

## Cohérence avec la suite "Mark"

Le logo doit s'intégrer visuellement avec:
- **Backmark** - Gestion de backlog (couleur: blue/purple)
- **Stackmark** - Docker/containers (couleur: blue/cyan)
- **Shipmark** - Releases (couleur: orange/amber)
- **EnvMark** - Variables d'env (couleur: emerald green)

Chaque logo de la suite partage:
- Style minimaliste similaire
- Forme carrée pour l'icône
- Même famille typographique
- Suffixe "mark" traité de la même façon
